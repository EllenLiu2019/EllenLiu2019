## 1. 从投票到智能体协作：面向BioASQ 14b的答案类型感知型大语言模型流水线

- **原文链接**: http://arxiv.org/abs/2607.06452v1
- **发布日期**: 2026-07-07
- **摘要**:

生物医学问答不仅需要从科学文献中准确提取信息，还需跨多篇文档可靠整合证据。本研究提出一种面向BioASQ 14b任务B的特定问题类型大语言模型框架，旨在提升生物医学问答中答案的鲁棒性与证据溯源能力。该框架并非对所有问题采用单一提示策略，而是根据是非题、事实题和列举题不同的推理与评估需求，选择差异化的推理流程。针对是非题，采用片段重排与自我反思机制降低对证据顺序的敏感性，提升决策稳定性；针对事实题，结合完整片段输入与基于思维链的上下文学习，支持准确的生物医学实体识别；针对列举题，则采用多智能体架构，由证据提取、候选生成、答案验证与最终聚合模块协同处理。通过BioASQ 13b的初步实验确定各类问题的最优推理策略，最终框架在官方BioASQ 14b任务B挑战中接受评估。官方评测显示，本框架在多批次中表现优异，并在第4批次事实题子任务中取得第一名。这些结果证明了结合特定问题类型推理、集成预测与基于智能体的验证方法对实现可靠生物医学问答的有效性。

## 2. Danus：基于事实图谱记忆的数学推理智能体编排系统

- **原文链接**: http://arxiv.org/abs/2607.06447v1
- **发布日期**: 2026-07-07
- **摘要**:

近期基于大语言模型的数学推理智能体已开始攻克研究级问题，并在多个案例中为开放问题的解决做出了贡献。然而，由于在协调并行证明搜索的同时保持中间声明的组织性和可靠性存在困难，有效扩展和编排此类智能体仍具挑战性。本文提出Danus系统——一种以共享事实图作为全局内存管理机制的研究级数学推理编排系统。Danus包含执行规划与协调的主智能体、并行执行证明搜索的多个工作智能体，以及在数学声明被纳入事实图前进行验证的无状态验证器。每个经过验证的事实将与其证明过程和逻辑依赖关系一同存储，使系统能够逐步构建长论证链，同时保持共享证明状态的有序性。主智能体会定期总结演进的证明状态，将工作智能体导向有前景的方向，并通过进度报告支持与人类数学家的交互。我们通过代数几何、奇点理论和组合学领域的六个研究级案例研究对Danus进行评估，展示了事实图内存机制如何使Danus能够构建冗长而严谨的数学证明。实验结果表明，基于事实图的编排为扩展面向长周期研究问题的数学推理智能体提供了有效路径。Danus已在https://github.com/frenzymath/Danus开源。

## 3. RuBench：一个基于原生俄语任务规范的仓库级智能编码基准

- **原文链接**: http://arxiv.org/abs/2607.06411v1
- **发布日期**: 2026-07-07
- **摘要**:

开发者越来越多地将实际维护工作委托给产品级编码智能体，并以母语描述任务，其风格类似客户请求而非精心编写的英文问题。现有仓库级智能体基准测试并未衡量这一场景：其任务描述均为英文设计。我们推出 RuBench 1.0 基准测试，包含从五个活跃开源仓库（aiohttp、aiogram、Laravel、NestJS、Fastify；涉及 Python、PHP、TypeScript、JavaScript）近期修复提交中挖掘的 25 个任务。每个任务均以俄语原生描述——完全以真实客户请求风格从零编写（非翻译版本）——并由上游维护者的回归测试进行评判，这些测试我们暂不公开。所有 25 个修复提交均晚于每个评估模型的训练数据截止日期，从而提供逐任务可验证的污染论证。我们对已部署的产品配置（CLI 智能体 + 模型 + 推理努力度）——Claude Code 搭配 Opus 4.8、Sonnet 5 和 Haiku 4.5，以及 Codex CLI 搭配 GPT-5.5——进行三次独立运行评估，报告带任务级置信区间的 pass@1、配对比较、美元成本及令牌使用量。最佳配置解决了 78.7% 的任务；在 N=25 时，仅与最弱模型之间的差距具有统计显著性，我们对此明确说明。通过审计第五个非竞赛配置（Claude Code + Fable 5，2026 年 7 月 2 日发布版本）的完整轨迹，我们发现产品在静默替换模型：在 25 个任务中的 5 个（20%）中，官方安全防护回退机制将常规 HTTP 协议修复重新路由至 Opus 4.8——这是直接、可复现的证据，表明实际测量单位是已部署的产品而非模型。我们公开任务描述、元数据、完整智能体轨迹及差异文件；评分标准暂不公开，但已在发布时提交 SHA-256 清单。
